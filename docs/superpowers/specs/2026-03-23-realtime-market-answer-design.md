# 实时行情问答设计

## 目标
当用户提问股票、基金、金价、油价、汇率、加密货币这类实时问题时，系统优先走实时数据通道，不再完全依赖大模型的训练数据记忆。命中后直接返回系统生成的实时答案；未命中继续走原有多模型问答链路。

## 范围
第一版覆盖：
- 股票：A 股、港股、美股常见标的
- 基金：常见基金代码与名称查询
- 金价：国际金价 + 人民币估算
- 油价：国际油价 + 国内油价摘要
- 汇率：主流货币对
- 加密货币：主流币种

不在第一版做：
- 深度 K 线图
- 技术指标
- 个股公告全文解析
- 高精度专业金融终端级数据

## 核心问题
当前项目已经有 `realtimeContextService`，但它只是把实时数据塞进 system prompt，让模型自己组织答案。这样有三个问题：
1. 模型可能忽略注入数据
2. 模型可能混入旧知识继续胡说
3. 实时类型不全，缺基金、油价

所以第一版要把“实时行情类问题”从提示增强，升级为“后端分流直答”。

## 总体架构
用户问题进入 `/api/chat` 后，先做一层实时意图识别。

- 未命中实时意图：继续现有多模型流程
- 命中实时意图：
  1. 识别查询类别与标的
  2. 拉取实时数据
  3. 标准化成统一结构
  4. 用系统模板直接生成答案
  5. 前端只显示 1 张“实时行情结果”卡片

如果实时数据失败：
- 按用户选择的策略，降级给大模型回答
- 但必须在答案中明确提示“实时数据获取失败，以下回答可能不是实时”

## 模块拆分
### 1. 实时意图识别层
新增 `backend/src/services/realtimeIntentService.ts`

职责：
- 判断当前问题是否属于实时行情问题
- 分类到 `stock | fund | gold | oil | forex | crypto`
- 提取代码、名称、市场、货币对等关键参数

输出统一结构：
- `matched`
- `kind`
- `target`
- `displayName`
- `confidence`

### 2. 实时行情拉取层
新增 `backend/src/services/realtimeQuoteService.ts`

职责：
- 根据类别调用不同免费公开源
- 封装超时、缓存、错误处理、字段归一化
- 输出统一 quote 结构

第一版建议数据源：
- 股票/金价：优先复用 Stooq 路径
- 汇率：继续用 ExchangeRate-API
- 加密货币：继续用 CoinGecko
- 油价：新增公开油价源或公开商品报价源
- 基金：新增免费基金净值/估值源

### 3. 实时答案格式化层
新增 `backend/src/services/realtimeAnswerFormatter.ts`

职责：
- 不依赖模型，自行拼接最终答案
- 输出统一 Markdown/纯文本
- 包含：
  - 标的名称
  - 当前价格/净值/汇率
  - 涨跌或波动信息（能拿到就显示）
  - 数据来源
  - 数据时间
  - 免责声明

### 4. 聊天路由分流层
修改 `backend/src/routes/chat.ts`

职责：
- 在现有 `buildRealtimeContextMessage` 前做更高优先级分流
- 命中实时意图时，短路原 SSE 多模型输出
- 返回统一 realtime payload

### 5. 前端展示层
修改 `frontend/src/views/ChatView.vue`

职责：
- 识别后端返回的 realtime payload
- 在多模型区只显示一张“实时行情结果”卡
- 样式上明确标记：实时 / 数据来源 / 更新时间
- 不再重复渲染多模型结果卡

## 返回协议
建议后端在命中实时路由时返回统一 meta：
- `mode: realtime_quote`
- `realtime.kind`
- `realtime.target`
- `realtime.source`
- `realtime.timestamp`
- `realtime.fallbackUsed`

正文内容仍然走现有前端容易接入的 `content` 字段，降低改动面。

## 缓存策略
- 行情类缓存：30-60 秒
- 新闻/油价摘要：60-180 秒
- 按 `kind + target` 做 key
- 请求失败不缓存成功值覆盖，但可短暂缓存失败状态 5-10 秒防止抖动打爆源站

## 降级策略
用户已确认：失败时继续让大模型回答，但要明确可能不是实时。

落地方式：
- 实时源成功：直接系统模板答
- 实时源失败：
  - 写入一个 system guardrail
  - 告诉模型“实时数据未获取成功，以下内容可能非实时，请明确说明”
  - 同时把失败原因记 observability

## 风险
1. 免费源稳定性一般，尤其基金、油价
2. 标的解析会有歧义，例如“平安”“招商”
3. 有些场外基金只有 T-1 净值，严格意义不算实时
4. 多语言/模糊问法会影响识别准确率

## 第一版取舍
- 先追求“命中后不胡说”
- 不追求所有标的 100% 覆盖
- 不追求专业金融终端精度
- 先把基金和油价补齐，并把现有黄金/股票/汇率/币价从“提示注入”升级成“后端直答”

## 验证标准
- 提问“今日金价”“当前油价”“腾讯股价”“美元兑人民币”“比特币现在多少钱”“天弘沪深300基金净值”时：
  - 命中实时路由
  - 返回来源和时间
  - 不出现模型胡编身份/旧知识口吻
- 数据源失败时：
  - 不崩
  - 会降级
  - 会明确提示可能非实时
